## 任务：每天自动监控 arXiv 论文并推送飞书

### 第一步：执行 Python 脚本
cd 到项目根目录后，用 python3 运行：
```bash
cd /path/to/hermes-arxiv-agent && python3 monitor.py
```

### 第二步：判断是否有新论文
运行完毕后检查脚本输出：
- 如果包含 `No new papers`，直接回复：`✅ 今日（YYYY-MM-DD）未发现新的论文。`
- 如果有 `LLM_SUMMARIZATION_REQUIRED` 部分，进入第三步。

### 第三步：读取 JSON 并完成 LLM 信息补全
读取 `/path/to/hermes-arxiv-agent/new_papers.json`，该文件包含 `new_count`、`pending_count`、`new_papers` 和 `papers_to_process`。

其中：
- `new_count` 表示本次新抓取的论文数
- `pending_count` 表示当前仍需要 agent 补全的论文数
- 优先以 `papers_to_process` 作为本次必须处理的论文列表；若不存在，再使用 `new_papers`

对待处理列表中的每篇论文必须完成以下两步：

**1）提取作者单位 affiliations**
- 使用 pdfplumber 或 PyMuPDF 读取 PDF（路径在 pdf_local_path 字段）前 2 页文本
- 优先使用上标脚注映射（如 ^1、^2 对应单位）
- 如无清晰脚注，再匹配常见机构名（MIT、Stanford、Google、Microsoft、Tsinghua、Peking 等）
- 对 CamelCase 连写词做分词还原：`DepartmentofPoliticalSciences` → `Department of Political Sciences`
- 合并跨行连字符词：`Repub-` + `licof Korea` → `Republic of Korea`
- 删除噪音：URL、正文叙述句、公式片段、参考文献片段
- 若含连续 18+ 个无空格英文字母，必须再次清洗
- 多个单位用 `;` 分隔，不要换行，不要保留脚注编号
- 无法确定时填：`未找到单位信息`

**2）生成中文总结 summary_cn**
- 基于 abstract 生成，90-150 个中文字符
- 必须覆盖：方法核心、主要贡献、关键结果
- 禁止模板化空话，禁止仅写"提出新型方法"等泛化句
- 只输出一段自然中文，不使用分点
- 质量自检：少于 90 字或与其他摘要高度雷同时重写

### 第四步：更新 Excel
将 affiliations 和 summary_cn 回填到 `/path/to/hermes-arxiv-agent/papers_record.xlsx`（按 arxiv_id 匹配）。

### 第五步：更新网站数据
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 viewer/build_data.py
```

确认 `viewer/papers_data.json` 已根据最新 Excel 重建，包含刚回填的 affiliations 和 summary_cn。

### 第六步：构建飞书 Markdown 消息
输出格式：
```
📚 **论文日报** | YYYY-MM-DD
共发现 **N** 篇新论文
[每篇：标题、arXiv ID、日期、作者、单位、PDF 链接、90-150 字中文摘要]
PDF 已下载至 papers/，记录已更新至 papers_record.xlsx，网站数据已更新至 viewer/papers_data.json。
```
**重要：最终回复只输出飞书消息文本，不要输出其他说明。**
